# What this module covers

The first module introduces [Boomi Companion](/concepts/boomi-companion.md) as an AI development partner for building Boomi integrations. It positions Companion not as a chatbot layered onto the platform, but as a collaborator that reads your intent, uses [skill folders](/concepts/skill-folders.md) to ground itself in Boomi platform knowledge, and produces real integration artifacts you can inspect on the canvas.

# Key ideas

- Boomi Companion is an agent that co-creates integrations with you, not a code generator you prompt in isolation.
- Companion's competence comes from skill folders that describe how to use the Boomi platform.
- The agent skill closes the gap between "I know what I want" and "I have a working integration".

# Framing

The course refers to this as the "agent skill closes the gap" idea: raw language models know what integrations are in the abstract, but only an agent grounded in Boomi-specific skill definitions can produce artifacts that actually work in your tenant.
